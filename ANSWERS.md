1) Event Propagation (Event Propagation в JavaScript)
 событие проходит через три фазы — захват (capture) сверху вниз, целевая (target) на элементе и всплытие (bubble) снизу вверх. Практика: делегирование событий (назначаем один обработчик на родителя), предотвращение дальнейшей обработки через event.stopPropagation(), и выбор нужной фазы через addEventListener(..., { capture: true }). Это уменьшает количество обработчиков и упрощает работу с динамическими элементами.

2) Promise и асинхронность
 Promise - объект, представляющий результат асинхронной операции (состояния: pending, fulfilled, rejected). Обрабатывается через .then/.catch/.finally или с async/await (синтаксический сахар). Альтернативы: callbacks, RxJS (observables). Event Loop управляет порядком выполнения: сначала синхронный код, затем микрозадачи (promises), потом макрозадачи (setTimeout, I/O) — это объясняет порядок вывода и реакции UI.

3) ООП (Объектно-ориентированное программирование)
 ООП строится вокруг объектов (свойства + методы). Ключевые принципы - инкапсуляция (скрытие состояния), наследование (extends), полиморфизм (один интерфейс - разные реализации) и абстракция. В JS ООП реализуется через прототипы и классы (ES6 - синтаксический сахар). Примеры: class Person { constructor(){}; greet(){} } и использование Object.create() для прототипного наследования.

4) Обработка URL браузером (схема)
 ввод URL → DNS-реквизит → TCP/TLS handshake → HTTP(S) запрос → сервер обрабатывает запрос → возвращает ответ → браузер парсит HTML → строит DOM и CSSOM → layout → paint → composite. Ускорение: CDN, preconnect/preload, HTTP/2/3, сжатие, кеширование, lazy-loading. Проблемы: XSS, CSRF, CORS (междоменные ограничения), mixed content (HTTP на HTTPS). Для безопасности применяют CSP, SameSite cookies, CORS-заголовки и заголовки защиты.
